Section 14

Build & Run Instructions

Prerequisites

  • GCC — GNU C Compiler (comes with MinGW on Windows, pre-installed on Linux/macOS)
  • Python 3 — For the web server (server.py)
  • Web Browser — Any modern browser (Chrome, Firefox, Edge)

Method 1: Run with Web Server (Full UI)

# Navigate to the project directory
cd ADS_unified/

# Start the web server (auto-compiles C code)
python server.py

# Output:
#   [OK] Compiled successfully -> ./stock_market
#   Open in browser: http://localhost:8000

# Open browser to http://localhost:8000

Method 2: Run CLI Only (No Server)

# Compile manually
gcc -o stock_market main.c member1_hashtable.c member2_priority_queue.c \
    member3_avltree.c member4_heap.c member5_trie.c -lm

# Run the binary
./stock_market

# Select modules 1-5 from the menu, or 6 to run all

Method 3: Using Makefile

make          # Compile all
./stock_market  # Run
make clean    # Clean build artifacts
Section 15

Conclusion

The StockADS — Stock Market Data Aggregator project successfully demonstrates 5 core Advanced Data Structures through a practical, real-world financial application:

Data StructureSyllabus UnitStock Market ApplicationKey Complexity
Hash TableFoundationO(1) stock price lookupO(1) avg insert/search
Priority QueueUnit 2Urgent-first alert processingO(log n) insert/extract
AVL TreeUnit 1Balanced time-series storageO(log n) guaranteed
HeapUnit 1, 2Top gainers/losers rankingO(log n) + O(1) peek
TrieUnit 3Symbol auto-completeO(m) independent of n

Each data structure was chosen because it uniquely solves a specific stock market problem that no other data structure can solve as efficiently:

  • Only a hash table gives O(1) amortized lookup by symbol
  • Only a priority queue ensures urgent-first processing regardless of arrival order
  • Only an AVL tree guarantees O(log n) worst-case for sorted time-series data
  • Only a dual heap enables simultaneous max and min extraction for rankings
  • Only a trie supports prefix-based auto-complete in O(m) time

The project features a three-layer architecture — C backend for algorithmic implementation, Python server for compilation and execution, and a premium HTML/CSS/JS frontend for visualization. Each data structure can be interacted with visually through the frontend or executed as raw C code via the "Run Live" feature.

Academic Significance

This project bridges the gap between theoretical ADS concepts and practical applications. Every insertion, rotation, heapification, and prefix search in the codebase directly maps to algorithms taught in the ADS syllabus. The modular architecture allows each team member to independently implement, test, and present their data structure while the unified system demonstrates how these structures work together in a real application.
