Topic 1: Rule-Based AI (Expert System)

A Rule-Based Expert System makes decisions using predefined IF-THEN rules — mimicking how a human expert would reason. In our system, the travel agent's expertise is encoded as production rules in recommender.py.

What is a Rule-Based Expert System?

An expert system has three components: (1) Knowledge Base — stores facts (our JSON dataset), (2) Inference Engine — applies rules to derive conclusions (our scoring logic), (3) Explanation Facility — explains why decisions were made (our XAI output).

Topic 2: Fuzzy Logic / Fuzzy Matching

Fuzzy Logic deals with degrees of truth rather than absolute True/False. In classical logic, "trek" ≠ "Trekking" is False. In fuzzy logic, it's "nearly True" — and our system handles this.

What is Fuzzy Matching?

Instead of requiring exact user input, the system uses approximate string matching (via Python's difflib library) to interpret imprecise input. This mimics how humans understand "close enough" matches.

Implementation — difflib.get_close_matches()

import difflib

def fuzzy_match_interest(user_input, trip_types, threshold=0.6):
    """
    Fuzzy match user interest against destination's trip types.
    Uses SequenceMatcher (Levenshtein-like) for approximate matching.
    
    Examples:
      "trek"   → matches "Trekking"     (substring match)
      "beech"  → matches "Beach"        (edit distance ≤ 2)
      "advntr" → matches "Adventure"    (close_matches similarity ≥ 0.6)
    """
    user_lower = user_input.lower()
    for trip_type in trip_types:
        type_lower = trip_type.lower().replace('_', ' ')
        
        # Method 1: Substring containment
        if user_lower in type_lower or type_lower in user_lower:
            return True
        
        # Method 2: Levenshtein distance (edit distance ≤ 2)
        if levenshtein(user_lower, type_lower) <= 2:
            return True
        
        # Method 3: difflib similarity ratio ≥ threshold
        matches = difflib.get_close_matches(user_lower, [type_lower], cutoff=threshold)
        if matches:
            return True
    
    return False

Fuzzy Matching in the Frontend (JavaScript)

// FROM index.html — Levenshtein distance implementation
function lev(a, b) {
    const m = a.length, n = b.length, d = [];
    for (let i = 0; i <= m; i++) d[i] = [i];
    for (let j = 0; j <= n; j++) d[0][j] = j;
    for (let i = 1; i <= m; i++)
        for (let j = 1; j <= n; j++)
            d[i][j] = a[i-1] === b[j-1]
                ? d[i-1][j-1]
                : 1 + Math.min(d[i-1][j], d[i][j-1], d[i-1][j-1]);
    return d[m][n];
}

// Fuzzy match: substring OR edit distance ≤ 2
function fuzzy(userInput, types) {
    const ul = userInput.toLowerCase();
    return types.some(t => {
        const tl = t.toLowerCase().replace(/_/g, ' ');
        return tl.includes(ul) || ul.includes(tl) || lev(ul, tl) <= 2;
    });
}

Topic 3: Heuristic AI Scoring System

A heuristic is an intelligent shortcut — a function that estimates the quality of a solution without exhaustive search. Our scoring engine uses a 7-parameter weighted heuristic to evaluate each of the 100 destinations.

The 7 Scoring Parameters

ParameterMax PointsWeightLogic
Budget Fit30Highest30 if within budget tier, 20 if luxury, 15 if mid-range, reject if below
Interest Match25High25 if fuzzy match found, reject if no match and interests specified
Season Suitability20High+20 if best season, +8 neutral, -15 if avoid season
Traveller Type15Medium+15 if ideal for this traveller, +3 otherwise

Complete Scoring Function

def score_place(place, budget, interests, season, traveller, days, region):
    """
    Heuristic evaluation function: h(n) = w₁·budget + w₂·interest + w₃·season 
                                         + w₄·traveller + w₅·duration + popularity + safety/2
    Returns: total score (int) or None if hard-constraint fails
    """
    score = 0
    
    # ═══ PARAMETER 1: BUDGET FIT (max 30 pts) ═══
    lo, hi = place.budget_range      # Budget tier range
    mlo, mhi = place.midrange        # Mid-range tier
    if lo <= budget <= hi:   score += 30    # Perfect budget fit
    elif mlo <= budget <= mhi: score += 15  # Mid-range fit
    elif budget < lo:          return None  # HARD REJECT
    else:                      score += 20  # Luxury fit
    
    # ═══ PARAMETER 2: INTEREST MATCH (max 25 pts) ═══
    if not interests:         score += 12  # No preference = generic bonus
    elif fuzzy_match(interests, place.trip_types):
                               score += 25  # Fuzzy match found!
    else:                      return None  # HARD REJECT
    
    # ═══ PARAMETER 3: SEASON SUITABILITY (max 20 pts) ═══
    if season:
        if season in place.best_seasons:    score += 20
        elif season in place.avoid_seasons: score -= 15
        else:                               score += 8
    
    # ═══ PARAMETER 4: TRAVELLER TYPE (max 15 pts) ═══
    if traveller:
        if traveller in place.ideal_for:   score += 15
        else:                               score += 3
    
    # ═══ PARAMETER 5: DURATION FIT (max 10 pts) ═══
    if days:
        if place.min_days <= days <= place.max_days: score += 10
        elif days < place.min_days:                   score -= 5
    
    # ═══ PARAMETER 6 & 7: POPULARITY + SAFETY ═══
    score += place.popularity_score         # +1 to +10
    score += place.safety_rating // 2      # +1 to +5
    
    return score

XAI Code — Generating Explanations

def print_card(place, score, traveller_type):
    """Generate explainable recommendation card with transparent reasoning."""
    
    # ─── XAI 1: Show the score transparently ───
    print(f"Score: {score}/~115")
    
    # ─── XAI 2: Explain season suitability ───
    if season in place.best_seasons:
        print("✓ BEST in this season — ideal time to visit")
    elif season in place.avoid_seasons:
        print("⚠ AVOID this season — not recommended")
    
    # ─── XAI 3: Explain WHY this suits the traveller ───
    if traveller_type and traveller_type in place.ideal_for_why:
        why = place.ideal_for_why[traveller_type]
        print(f"Why this suits you: {why}")
    
    # ─── XAI 4: Safety & Permit transparency ───
    if place.permits_required:
        print(f"⚠ PERMIT REQUIRED: {place.permit_details}")
    print(f"Safety: {place.safety_rating}/10")

Why XAI Matters

Traditional AI systems (especially neural networks) are "black boxes" — you get an answer but don't know why. Our Expert System is inherently explainable because every decision traces back to specific rules and data. This is a key advantage of Symbolic AI over Sub-Symbolic (neural) AI, and is a critical requirement in fields like healthcare, finance, and governance.

Section 15

Conclusion

The Yatra India AI Travel Recommender successfully demonstrates that Artificial Intelligence is not limited to machine learning. Using Symbolic AI techniques — the original and foundational approach to AI — we built a complete intelligent system that:

  • Reasons about travel decisions using production rules (Rule-Based Expert System)
  • Handles imprecise user input through fuzzy string matching (Fuzzy Logic)
  • Evaluates and ranks destinations using a weighted heuristic function (Heuristic Search)
  • Stores domain knowledge in a structured symbolic format (Knowledge Representation)
  • Explains every decision in human-readable language (Explainable AI)

Each concept maps directly to the AI syllabus units, demonstrating practical, real-world application of theoretical AI principles in a travel recommendation domain.

Key Takeaway

This project proves that meaningful AI applications can be built without neural networks, GPUs, or cloud APIs. Rule-based reasoning, heuristic search, and knowledge representation — the foundations of AI — are powerful enough to solve real problems when properly designed and implemented.
