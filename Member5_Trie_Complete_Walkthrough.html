Section 01

What is a Trie? — The Big Picture

A Trie (pronounced "try") comes from the word reTRIEval. It's a tree-like data structure where each node represents a single character. Paths from the root to marked nodes spell out complete words/keys.

🧠 Simple Analogy — Phone Ka Contacts

Jaise aapke phone mein "A" type karo to Amit, Ankit, Aman sab suggest hota hai — wahi kaam Trie karta hai! Trie ek tree hai jismein har node ek letter store karta hai. Root se kisi bhi leaf tak ka path ek complete word banata hai.

Visual Example — How "AAPL", "AMD", "AMZN" are stored:

[ROOT] | [A] ← First letter of all 3 words / \ [A] [M] ← Second letter branches | / \ [P] [D] [Z] ← AMD ends here (is_end=1) | | [L] [N] ← AAPL ends here, AMZN ends here Path ROOT→A→A→P→L = "AAPL" ✓ Path ROOT→A→M→D = "AMD" ✓ Path ROOT→A→M→Z→N = "AMZN" ✓

Key Properties of a Trie

PropertyDescription
Root NodeAlways empty — represents the starting point, no character stored in it
Each EdgeRepresents one character (A-Z in our case = 26 possible children)
Each NodeHas an array of 26 child pointers (one per letter A through Z)
is_end flagMarks whether a complete word ends at this node (1 = yes, 0 = no)
Shared Prefixes"AAPL" and "AMD" share the "A" node — memory efficient for common prefixes!
No HashingUnlike hash tables, tries don't use hash functions — they use character-by-character navigation
⚡ Trie vs Hash Table vs Binary Search Tree

Hash Table: O(1) lookup but CANNOT do prefix search. "A" se start hone wale sab stocks nahi mil sakte.
BST/AVL: O(log n) lookup, can do range queries but not prefix matching efficiently.
Trie: O(m) lookup where m = key length. CAN do prefix search! Type "A" and instantly get all stocks starting with A. This is WHY we use Trie for auto-complete.

Line 31: char_idx() — Character to Index Mapping

static int char_idx(char c) { return toupper(c) - 'A'; }
Line 31
static int char_idx(char c) { return toupper(c) - 'A'; }
This is the MAPPING function — converts a letter to an array index (0-25).

How it works step by step:
1. toupper(c) — converts character to uppercase. 'a'→'A', 'B'→'B' (already uppercase stays same)
2. - 'A' — subtracts ASCII value of 'A' (which is 65)

Examples:
• char_idx('A') = toupper('A') - 'A' = 65 - 65 = 0 → children[0]
• char_idx('B') = toupper('B') - 'A' = 66 - 65 = 1 → children[1]
• char_idx('M') = toupper('M') - 'A' = 77 - 65 = 12 → children[12]
• char_idx('Z') = toupper('Z') - 'A' = 90 - 65 = 25 → children[25]

This is a one-liner function — the entire function body is on one line with curly braces. It's equivalent to writing it in 3 lines.

Faculty Answer: "char_idx maps a character to its corresponding array index using ASCII arithmetic. toupper() ensures case-insensitivity, and subtracting 'A' maps the uppercase letter to the range 0-25, which directly indexes into the children array."
💡 ASCII Value Table (Important for Viva)

'A' = 65, 'B' = 66, 'C' = 67 ... 'Z' = 90
'a' = 97, 'b' = 98, 'c' = 99 ... 'z' = 122
'0' = 48, '1' = 49 ... '9' = 57

So 'M' - 'A' = 77 - 65 = 12. This trick works because ASCII values for letters are contiguous (no gaps between A and Z).

This function inserts a stock symbol into the trie, character by character. This is THE core operation.

static void trie_insert(TrieNode* root, const char* symbol, double price) {
    TrieNode* cur = root;
    printf("  [INSERT] %s ($%.2f)\n", symbol, price);
    for(int i = 0; symbol[i]; i++) {
        int idx = char_idx(symbol[i]);
        if(idx < 0 || idx >= ALPHABET_SIZE) continue;
        if(!cur->children[idx]) cur->children[idx] = trie_new_node();
        cur = cur->children[idx];
    }
    cur->is_end = 1;
    strncpy(cur->symbol, symbol, MAX_SYMBOL_LEN - 1);
    cur->last_price = price;
    trie_words++;
}

Lines 48–56: trie_search() — Exact Lookup

This function checks if a COMPLETE symbol exists in the trie. Returns 1 (true) if found, 0 (false) if not.

static int trie_search(TrieNode* root, const char* symbol) {
    TrieNode* cur = root;
    for(int i = 0; symbol[i]; i++) {
        int idx = char_idx(symbol[i]);
        if(idx < 0 || idx >= ALPHABET_SIZE || !cur->children[idx]) return 0;
        cur = cur->children[idx];
    }
    return cur && cur->is_end;
}

THIS LINE IS TRICKY — FACULTY FAVORITE! 🎯

After the loop, cur points to the node of the last character. But we need TWO checks:

1. cur — is the pointer not NULL? (safety check)
2. cur->is_end — is this node marked as end of a word?

WHY do we need is_end check?

- But cur->is_end is 0 at M node (because "AM" is NOT a valid stock symbol)

Complexity Analysis & Space-Time Tradeoff

OperationTimeExplanation
InsertO(m)m = length of key. Walk/create one node per character. "AAPL" = 4 steps.
Exact SearchO(m)Walk the trie for m characters. Check is_end at the final node.
Auto-CompleteO(m + r)m steps to reach prefix node + r steps to collect r results via DFS.
Space (per node)O(26) = O(1)Each node has a fixed-size array of 26 pointers regardless of usage.
Total SpaceO(26 × N)N = total nodes. Each node stores 26 pointers. With 58 nodes → ~13KB.
⚖️ The Space-Time Tradeoff

Trie WASTES space — each node has 26 child pointers, but most are NULL. A typical node might use only 1-2 of its 26 slots.

But it GAINS speed — O(m) lookup independent of how many words exist. A hash table is O(1) average but CANNOT do prefix search. You'd need to check every key!

Real numbers: 18 stocks, 58 nodes, ~13KB memory. The same 18 stocks in a hash table would use ~2KB. But the hash table can't answer "give me all stocks starting with A" without scanning everything.

Conclusion: Trie trades ~6x more memory for the ABILITY to do instant prefix queries. For auto-complete, this is worth it.

Faculty FAQ — Tricky Questions & Answers 🎯

Q1
Why did you use calloc instead of malloc?
calloc zero-initializes all memory. This sets all 26 child pointers to NULL automatically. With malloc, we'd need a loop to set each pointer to NULL manually. calloc is cleaner and safer for structures with pointer arrays.
Q2
What does "static" mean before a function?
Static limits the function's visibility to the current file (translation unit). Other .c files cannot call static functions. It's like "private" in Java/C++. Only run_trie() is non-static (public) — it's the module's entry point.
Q3
Why is ALPHABET_SIZE 26? What if we had digits too?
26 for uppercase A-Z only. If we needed digits (0-9), we'd change to 36. If we needed full ASCII, 128. But more children per node = more memory per node = less efficient. Since stock symbols are only letters, 26 is optimal.
Q4
What does "symbol[i]" in the for-loop condition mean?
In C, strings end with '\0' (null terminator, ASCII value 0). The loop condition symbol[i] evaluates the character — any non-zero character is "true" (continue), but '\0' is "false" (stop). This is a standard C idiom for iterating through strings.
Q5
Why use strncpy instead of strcpy?
strcpy has NO length limit — if the source string is too long, it overflows the destination buffer (security vulnerability!). strncpy takes a max length parameter, copying at most N-1 characters. This prevents buffer overflow attacks. It's safer.
Q6
What is the difference between trie_search and get_suggestions?
trie_search checks if an EXACT complete word exists (returns 0 or 1). get_suggestions finds ALL words that START WITH a given prefix. Search checks is_end at the final node. Suggestions navigates to the prefix node and then does DFS to collect all is_end nodes below.
Q7
Why is cnt passed as int* (pointer) in collect()?
Because collect() is recursive. If cnt was a regular int, each recursive call would get its OWN COPY, and modifications wouldn't propagate back to the caller. With a pointer, all recursive calls share the SAME counter variable, so everyone sees the updated count.
Q8
What happens if I search for "AM" when only "AMD" and "AMZN" exist?
trie_search("AM") returns 0 (NOT FOUND). The path A→M exists, but the M node has is_end = 0 because "AM" was never inserted as a complete word. However, get_suggestions("AM") would return AMD and AMZN — because they're complete words BELOW the AM prefix node.
Q9
What is toupper() and why do we need it?
toupper() from ctype.h converts lowercase to uppercase ('a'→'A'). We use it so our trie is case-insensitive — "aapl", "Aapl", "AAPL" all map to the same path. Without it, 'a' and 'A' would map to different indices (97-65=32 vs 65-65=0).
Q10
What is sizeof(stocks)/sizeof(stocks[0])?
A compile-time trick to calculate array length. sizeof(stocks) gives total bytes of the array. sizeof(stocks[0]) gives bytes of one element. Division = number of elements. This is better than hardcoding "18" because it auto-adjusts if you add/remove items.
