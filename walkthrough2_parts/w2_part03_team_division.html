Member 2 — Priority Queue (Min-Heap)
Priority Queue · Min-Heap · Heapify Up/Down · Alert Processing

What You Explain:

  • What is a Priority Queue? — Elements dequeued by priority, not arrival order
  • Why use a min-heap? — Parent ≤ children ensures the most urgent alert is at root
  • heapify_up — after insert, bubble the new element up to its correct position
  • heapify_down — after extract, push the replacement down to restore heap
  • Parent/child index formulas: parent = (i-1)/2, left = 2i+1, right = 2i+2
  • Show 4 priority levels: CRITICAL(1) → HIGH(2) → MEDIUM(3) → LOW(4)

3
Member 3 — AVL Tree (Self-Balancing BST)
AVL Tree · Balance Factor · LL/RR/LR/RL Rotations · O(log n) Guarantee

What You Explain:

  • What is an AVL Tree? — A BST where |height(left) - height(right)| ≤ 1 at every node
  • Balance Factor = height(left subtree) - height(right subtree)
  • 4 rotation cases: LL (right rotate), RR (left rotate), LR (left-right), RL (right-left)
  • Why rotations are O(1) but guarantee O(log n) tree height
  • Show avl_insert() with the 4-case rotation logic
  • Compare to plain BST: sequential inserts create O(n) linked list vs O(log n) AVL

Unit 1 (AVL Trees, Red-Black Trees, B-Trees, Splay Trees — all self-balancing tree variants)

Member 4 — Heap (Max/Min Ranking)
Heap · Max-Heap · Min-Heap · DEPQ · Top-K Extraction

What You Explain:

  • What is a Heap? — A complete binary tree satisfying the heap property
  • Max-Heap for gainers (root = biggest gainer) vs Min-Heap for losers (root = biggest loser)
  • This implements a Double Ended Priority Queue (DEPQ) — extract max AND min
  • How the same heap_insert() handles both modes with an is_max_heap flag
  • Array-based complete binary tree — no pointers needed, parent/child via index math
  • Top-K extraction: extract the root K times to get K best/worst stocks

5
Member 5 — Trie (Prefix Tree)
Trie · 26-ary Branching · Prefix Search · Auto-Complete · DFS Collection

What You Explain:

  • What is a Trie? — A tree where each node represents one character; paths spell words
  • 26-ary branching — each node has up to 26 children (A-Z)
  • Insert: walk character by character, create nodes as needed, mark is_end
  • Auto-complete: navigate to prefix node, then DFS to collect all descendants
  • O(m) search — independent of number of stored words (unlike hash table for prefix queries)
  • Space-time tradeoff: 26 pointers per node uses more memory but enables prefix matching
