Trie — Prefix Tree

What Is It? (Explain to Faculty)

A Trie (from retrieval, pronounced "try") is a tree-like data structure optimized for string operations. Each node represents a single character, and paths from root to leaf spell out stored keys. Unlike hash tables, tries support prefix queries — finding all words that start with a given prefix — in O(m + r) time, where m is the prefix length and r is the number of results.

In our project, the trie enables instant auto-complete: type "A" and immediately get AAPL, AMD, AMZN, ADBE, AVGO.

Where Is It Applied? (Code)

Trie Node — 26-ary Branching

// File: member5_trie.c
// Each node has 26 children — one for each letter A-Z

typedef struct TrieNode {
    struct TrieNode* children[26];  // 26-ary branching
    int    is_end;                   // 1 = complete word ends here
    double price;                    // Stock price at end node
    char   symbol[MAX_SYMBOL_LEN];   // Full symbol stored at end
} TrieNode;

Explain to faculty: "Each node has a 26-element pointer array — one slot per letter. To go from 'A' to 'P', we follow children[15] (P is the 16th letter, index 15). The is_end flag marks complete words. This 26-ary branching is the key design decision — it trades space for speed."

Auto-Complete — Prefix Search + DFS Collection

// File: member5_trie.c → get_suggestions()
// Step 1: Navigate to the prefix node
// Step 2: DFS to collect ALL complete words below that node

void get_suggestions(TrieNode* root, const char* prefix, ...) {
    TrieNode* curr = root;

    // Step 1: Walk to prefix node — O(m)
    for(int i = 0; prefix[i]; i++) {
        int idx = toupper(prefix[i]) - 'A';
        if(!curr->children[idx]) return;  // Prefix not in trie
        curr = curr->children[idx];
    }

    // Step 2: DFS collect all end-nodes — O(r)
    collect(curr, results, prices, count, max_results);
}

static void collect(TrieNode* node, ...) {
    if(node->is_end) {
        // Found a complete word — add to results
        strcpy(results[*count], node->symbol);
        (*count)++;
    }
    // Recurse through all 26 children
    for(int i = 0; i < 26; i++)
        if(node->children[i])
            collect(node->children[i], ...);
}

Explain to faculty: "Auto-complete works in two phases. Phase 1: Navigate to the prefix node by following each character — this is O(m) where m = prefix length. Phase 2: Run DFS from that node to collect all complete words below — this visits only relevant subtree nodes. For prefix 'A', we reach the 'A' node and then DFS finds AAPL, AMD, AMZN, ADBE, AVGO."
