Heap — Max/Min Ranking

What Is It? (Explain to Faculty)

A Heap is a complete binary tree stored as an array where every parent satisfies the heap property. In a Max-Heap, every parent ≥ its children (root = maximum). In a Min-Heap, every parent ≤ its children (root = minimum).

Our project uses both simultaneously: a max-heap ranks top gainers and a min-heap ranks top losers. This combination implements a Double Ended Priority Queue (DEPQ) — extracting both the maximum and minimum efficiently.

Where Is It Applied? (Code)

Dual-Mode Heap Insert

// File: member4_heap.c → heap_insert()
// Same function handles BOTH max-heap and min-heap via is_max_heap flag

void heap_insert(Heap* h, const char* sym, double price, double chg, long vol) {
    int i = h->size++;

    // Bubble up — direction depends on heap type
    while(i > 0) {
        int parent = (i - 1) / 2;
        if(h->is_max_heap) {
            // MAX-HEAP: parent should be >= child
            if(h->data[parent].change < h->data[i].change)
                swap(&h->data[parent], &h->data[i]);
            else break;
        } else {
            // MIN-HEAP: parent should be <= child
            if(h->data[parent].change > h->data[i].change)
                swap(&h->data[parent], &h->data[i]);
            else break;
        }
        i = parent;
    }
}

Explain to faculty: "The is_max_heap flag controls the comparison direction. For the gainer heap, we want the biggest change_percent at the root (max-heap). For the loser heap, we want the most negative change at the root (min-heap). The same heapify logic works for both — only the comparison operator changes. This is an elegant code reuse pattern."
