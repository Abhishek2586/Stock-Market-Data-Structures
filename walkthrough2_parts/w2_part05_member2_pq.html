Section 05 — Member 2

Priority Queue — Min-Heap

What Is It? (Explain to Faculty)

A Priority Queue is an abstract data type where each element has a priority. Unlike a regular queue (FIFO), elements are dequeued in priority order — the highest-priority element comes out first, regardless of when it was inserted.

We implement it using a Min-Heap — a complete binary tree where every parent node has a priority value ≤ its children. The root always holds the minimum (most urgent) element.

Where Is It Applied? (Code)

Heapify Up — Restoring Heap After Insert

// File: member2_priority_queue.c → heapify_up()
// After inserting at the end, bubble the element UP until heap property holds

void heapify_up(PriorityQueue* pq, int idx) {
    while(idx > 0) {
        int parent = (idx - 1) / 2;          // Parent index formula
        if(pq->heap[parent].priority > pq->heap[idx].priority) {
            swap(&pq->heap[parent], &pq->heap[idx]);  // Parent is bigger → swap
            idx = parent;                               // Continue up
        } else break;                               // Heap property satisfied
    }
}

Explain to faculty: "After inserting a new alert at the end of the array, heapify_up compares it with its parent. If the new element has higher urgency (lower priority number), it swaps upward. This continues until the root is reached or the heap property is satisfied. This is O(log n) — the height of the tree."

Heapify Down — Restoring Heap After Extract

// File: member2_priority_queue.c → heapify_down()
// After extracting root and moving last element to root, push it DOWN

void heapify_down(PriorityQueue* pq, int idx) {
    int size = pq->size;
    while(1) {
        int smallest = idx;
        int left  = 2 * idx + 1;              // Left child index
        int right = 2 * idx + 2;              // Right child index

        if(left < size && pq->heap[left].priority < pq->heap[smallest].priority)
            smallest = left;
        if(right < size && pq->heap[right].priority < pq->heap[smallest].priority)
            smallest = right;

        if(smallest == idx) break;           // Heap property satisfied
        swap(&pq->heap[idx], &pq->heap[smallest]);
        idx = smallest;
    }
}

Explain to faculty: "After extracting the root (most urgent alert), we move the last element to the root position. heapify_down then pushes it to the correct position by comparing with both children and swapping with the smaller child. This maintains the min-heap invariant in O(log n)."
