Topic 1: Hash Table with Chaining

A Hash Table is a data structure that maps keys to values using a hash function. It provides O(1) average-time insert, search, and delete operations. In our project, stock symbols (like "AAPL") are hashed to bucket indices for instant price lookups.

What is a Hash Table?

A hash table uses a hash function to compute an index into an array of buckets. When two keys hash to the same index (a collision), we use separate chaining — each bucket contains a linked list of all entries that hash to that index.

The djb2 Hash Function

// FROM member1_hashtable.c
// Daniel J. Bernstein's hash function — simple, fast, good distribution
unsigned int hash_function(const char* str) {
    unsigned long hash = 5381;        // Magic seed value
    int c;
    while ((c = *str++))
        hash = ((hash << 5) + hash) + c;  // hash * 33 + c
    return hash % TABLE_SIZE;          // Mod to fit in table
}

How djb2 works: Start with seed 5381, for each character multiply current hash by 33 (via bit shift) and add the character's ASCII value. Finally, modulo by TABLE_SIZE (100) to get the bucket index.

Collision Handling — Separate Chaining

// FROM member1_hashtable.c → ht_insert()
void ht_insert(HashTable* ht, const char* symbol, double price, long volume) {
    unsigned int index = hash_function(symbol);   // Compute bucket index
    Stock* current = ht->buckets[index];

    // Check if symbol already exists (UPDATE case)
    while(current != NULL) {
        if(strcmp(current->symbol, symbol) == 0) {
            current->price = price;                // Update existing entry
            current->change_percent = ((price - old_price) / old_price) * 100.0;
            return;
        }
        current = current->next;                   // Traverse chain
    }

    // New entry — prepend to chain (collision = chain grows)
    Stock* new_stock = (Stock*)malloc(sizeof(Stock));
    strcpy(new_stock->symbol, symbol);
    new_stock->price = price;

    if(ht->buckets[index] != NULL)
        ht_stats.collisions++;                     // COLLISION detected!

    new_stock->next = ht->buckets[index];          // Prepend to chain
    ht->buckets[index] = new_stock;
    ht->total_stocks++;
}

Search — O(1) Average

// FROM member1_hashtable.c → ht_search()
Stock* ht_search(HashTable* ht, const char* symbol) {
    unsigned int index = hash_function(symbol);   // Hash to find bucket
    Stock* current = ht->buckets[index];
    int chain_length = 0;

    while(current != NULL) {
        chain_length++;
        if(strcmp(current->symbol, symbol) == 0) {
            return current;                        // FOUND at chain position
        }
        current = current->next;
    }
    return NULL;                                   // NOT FOUND
}

Topic 2: Priority Queue — Min-Heap

A Priority Queue is an abstract data type where each element has a priority. Elements are dequeued in priority order, not insertion order. Our implementation uses a Min-Heap — the element with the lowest priority number (highest urgency) is always at the top.

What is a Priority Queue?

Unlike a regular FIFO queue, a priority queue ensures that the most urgent element is always processed first. In our stock market system, a CRITICAL trading halt (priority 1) must be processed before a routine price tick (priority 4), even if the price tick arrived first.

Priority Levels

PriorityLevelExample AlertColor
1CRITICALTrading HALT, Circuit breaker -7%Red
2HIGHVolume spike +300%, Index rebalanceOrange
3MEDIUMEarnings update, Analyst downgradeYellow
4LOWRoutine price tickGreen

Heapify Up — Restoring Heap Property After Insert

// FROM member2_priority_queue.c
// After inserting at the end, bubble up to restore min-heap property
static void heapify_up(PriorityQueue* pq, int idx) {
    while(idx > 0) {
        int par = pq_parent(idx);              // parent = (i-1)/2
        // Lower priority NUMBER = higher URGENCY = should be at top
        if(pq->data[par].priority > pq->data[idx].priority) {
            pq_swap(&pq->data[par], &pq->data[idx]);
            idx = par;                             // Continue up
        } else {
            break;                                // Heap property satisfied
        }
    }
}

Heapify Down — Restoring Heap Property After Extract

// After extracting root (min), move last element to root and push down
static void heapify_down(PriorityQueue* pq, int idx) {
    int smallest = idx;
    int left  = pq_left(idx);                  // 2*i + 1
    int right = pq_right(idx);                 // 2*i + 2

    if(left  < pq->size && pq->data[left].priority  < pq->data[smallest].priority)
        smallest = left;
    if(right < pq->size && pq->data[right].priority < pq->data[smallest].priority)
        smallest = right;

    if(smallest != idx) {
        pq_swap(&pq->data[idx], &pq->data[smallest]);
        heapify_down(pq, smallest);                // Recurse down
    }
}

Topic 3: AVL Tree — Self-Balancing BST

An AVL Tree (Adelson-Velsky and Landis) is a self-balancing Binary Search Tree where the height difference between left and right subtrees of any node is at most 1. This guarantees O(log n) worst-case time for insert, search, and delete — unlike a regular BST which can degrade to O(n).

Why AVL Trees Matter

In our stock market application, historical price records are inserted in timestamp order. A regular BST would degenerate into a linked list (all nodes on one side). The AVL tree's automatic rotations keep the tree balanced, ensuring fast range queries on historical data.

Balance Factor

// Balance Factor = height(left subtree) - height(right subtree)
// Valid values: -1, 0, +1
// If |BF| > 1 → IMBALANCED → rotation needed

int avl_balance(PriceNode* node) {
    if(!node) return 0;
    return avl_height(node->left) - avl_height(node->right);
}

The Four Rotation Cases

ImbalanceConditionRotationDescription
LLBF > 1 && key < left→keyRight RotationLeft-heavy, new node in left-left subtree
RRBF < -1 && key > right→keyLeft RotationRight-heavy, new node in right-right subtree
LRBF > 1 && key > left→keyLeft-Right (double)Left-heavy, new node in left-right subtree
RLBF < -1 && key < right→keyRight-Left (double)Right-heavy, new node in right-left subtree

Rotation Implementation

// FROM member3_avltree.c — Right Rotation (for LL imbalance)
PriceNode* rotate_right(PriceNode* y) {
    PriceNode* x  = y->left;        // x becomes new root
    PriceNode* T2 = x->right;       // T2 is x's right subtree

    x->right = y;                   // y becomes x's right child
    y->left  = T2;                  // T2 goes to y's left

    // Update heights
    y->height = 1 + MAX(avl_height(y->left), avl_height(y->right));
    x->height = 1 + MAX(avl_height(x->left), avl_height(x->right));

    return x;                       // Return new root
}

// Left Rotation (for RR imbalance) — mirror of right rotation
PriceNode* rotate_left(PriceNode* x) {
    PriceNode* y  = x->right;
    PriceNode* T2 = y->left;

    y->left  = x;
    x->right = T2;

    x->height = 1 + MAX(avl_height(x->left), avl_height(x->right));
    y->height = 1 + MAX(avl_height(y->left), avl_height(y->right));

    return y;
}

AVL Insert — Auto-Balancing

// FROM member3_avltree.c — Insert with automatic rebalancing
PriceNode* avl_insert(PriceNode* node, double price, long vol, int ts) {
    // 1. Standard BST insert
    if(!node) return create_price_node(price, vol, ts);
    if(ts < node->timestamp)   node->left  = avl_insert(node->left, price, vol, ts);
    else if(ts > node->timestamp) node->right = avl_insert(node->right, price, vol, ts);
    else return node;

    // 2. Update height
    node->height = 1 + MAX(avl_height(node->left), avl_height(node->right));

    // 3. Get balance factor
    int balance = avl_balance(node);

    // 4. Rotation cases
    if(balance >  1  && ts < node->left->timestamp)   return rotate_right(node);  // LL
    if(balance < -1  && ts > node->right->timestamp)  return rotate_left(node);   // RR
    if(balance >  1  && ts > node->left->timestamp)  {                            // LR
        node->left = rotate_left(node->left);
        return rotate_right(node);
    }
    if(balance < -1  && ts < node->right->timestamp) {                            // RL
        node->right = rotate_right(node->right);
        return rotate_left(node);
    }

    return node;
}

AVL trees are the strictest balanced BST (height difference ≤ 1), making them ideal for demonstrating self-balancing concepts before introducing relaxed balancing in Red-Black trees or multi-way branching in B-Trees.

Section 10

Topic 4: Heap — Max/Min Ranking

A Heap is a complete binary tree where each parent node satisfies the heap property. A Max-Heap has every parent ≥ its children (largest at root). A Min-Heap has every parent ≤ its children (smallest at root). Our project uses both simultaneously — a max-heap for top gainers and a min-heap for top losers.

Dual Heap Architecture

Stock arrives with change_percent: ┌─────────────────────────────────────────────┐ │ if change_percent >= 0 → MAX-HEAP (Gainers)│ Root = highest gainer │ if change_percent < 0 → MIN-HEAP (Losers) │ Root = biggest loser └─────────────────────────────────────────────┘ Extract-Max from Gainer Heap → "Top Gainer" (e.g., TSLA +12.3%) Extract-Min from Loser Heap → "Top Loser" (e.g., GOOGL -5.2%)

Heap Insert — Heapify Up

// FROM member4_heap.c
void heap_insert(Heap* h, const char* sym, double price, double chg, long vol) {
    if(h->size >= MAX_HEAP_SIZE) return;

    // Place at end of array
    int i = h->size++;
    strcpy(h->data[i].symbol, sym);
    h->data[i].price = price;
    h->data[i].change_percent = chg;

    // Bubble up: compare with parent, swap if heap property violated
    while(i > 0) {
        int parent = (i - 1) / 2;
        if(h->is_max_heap) {
            // MAX-HEAP: parent should be >= child
            if(h->data[parent].change_percent < h->data[i].change_percent) {
                heap_swap(&h->data[parent], &h->data[i]);
                i = parent;
            } else break;
        } else {
            // MIN-HEAP: parent should be <= child
            if(h->data[parent].change_percent > h->data[i].change_percent) {
                heap_swap(&h->data[parent], &h->data[i]);
                i = parent;
            } else break;
        }
    }
}

Heap Extract — Heapify Down

// FROM member4_heap.c — Extract root (max or min) and restore heap
HeapStock heap_extract(Heap* h) {
    HeapStock top = h->data[0];               // Save root (best element)
    h->data[0] = h->data[--h->size];           // Move last to root

    // Push down: compare with children, swap with appropriate child
    int i = 0;
    while(1) {
        int best = i;
        int left  = 2*i + 1;
        int right = 2*i + 2;

        if(h->is_max_heap) {
            if(left  < h->size && h->data[left].change_percent  > h->data[best].change_percent)  best = left;
            if(right < h->size && h->data[right].change_percent > h->data[best].change_percent) best = right;
        } else {
            if(left  < h->size && h->data[left].change_percent  < h->data[best].change_percent)  best = left;
            if(right < h->size && h->data[right].change_percent < h->data[best].change_percent) best = right;
        }

        if(best == i) break;
        heap_swap(&h->data[i], &h->data[best]);
        i = best;
    }
    return top;
}
Syllabus Connection — Unit 2: Heaps & DEPQs

Heaps (Unit 1, 2) — The dual-heap approach implements a Double Ended Priority Queue (DEPQ) concept from Unit 2. The max-heap extracts the maximum (top gainer), the min-heap extracts the minimum (top loser). This connects to Min-Max Heaps, Deaps, Leftist Trees, and Binomial Heaps from the syllabus — all advanced variations of the basic heap structure.

Topic 5: Trie — Prefix Tree Auto-Complete

A Trie (pronounced "try", from retrieval) is a tree-like data structure used for efficient string storage and retrieval. Each node represents a single character, and paths from root to leaf spell out stored keys. Our trie enables instant auto-complete — type "A" and get AAPL, AMD, AMZN, ADBE, AVGO in O(m + results) time.

Trie Insert — Character by Character

// FROM member5_trie.c
void trie_insert(TrieNode* root, const char* symbol, double price) {
    TrieNode* curr = root;
    for(int i = 0; symbol[i]; i++) {
        int idx = toupper(symbol[i]) - 'A';  // Map char to 0-25
        if(!curr->children[idx])
            curr->children[idx] = create_trie_node();
        curr = curr->children[idx];
    }
    curr->is_end = 1;                        // Mark as complete word
    strcpy(curr->symbol, symbol);
    curr->price = price;
}

Auto-Complete — Prefix Search + Collect All Descendants

// FROM member5_trie.c — Get all symbols matching a prefix
void get_suggestions(TrieNode* root, const char* prefix,
                     char results[][MAX_SYMBOL_LEN], double prices[],
                     int* count, int max_results) {
    TrieNode* curr = root;

    // Step 1: Navigate to the prefix node
    for(int i = 0; prefix[i]; i++) {
        int idx = toupper(prefix[i]) - 'A';
        if(!curr->children[idx]) return;  // Prefix not in trie
        curr = curr->children[idx];
    }

    // Step 2: Collect all complete words below this node
    collect(curr, results, prices, count, max_results);
}

// Recursive DFS to collect all end-nodes
static void collect(TrieNode* node, ...) {
    if(node->is_end) {
        strcpy(results[*count], node->symbol);
        prices[*count] = node->price;
        (*count)++;
    }
    for(int i = 0; i < 26; i++)
        if(node->children[i])
            collect(node->children[i], ...);    // DFS through all branches
}

Complexity Analysis

OperationTime ComplexityExplanation
InsertO(m)m = length of key; traverse/create one node per character
Exact SearchO(m)Traverse m nodes, check is_end flag
Auto-CompleteO(m + r)m to reach prefix node, r = number of results collected
SpaceO(26 × N)N = total nodes; each node has 26 child pointers
Space-Time Tradeoff

The trie uses 26 pointers per node (one per letter), which consumes more memory than a hash table. However, this "wasted" space enables prefix queries that hash tables cannot perform. With 18 symbols and 58 nodes, the trie uses approximately 11.8KB — a reasonable tradeoff for instant auto-complete capability.
