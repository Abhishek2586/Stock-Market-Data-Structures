Section 01

What is a Trie? — The Big Picture

A Trie (pronounced "try") comes from the word reTRIEval. It's a tree-like data structure where each node represents a single character. Paths from the root to marked nodes spell out complete words/keys.

🧠 Simple Analogy — Phone Ka Contacts

Jaise aapke phone mein "A" type karo to Amit, Ankit, Aman sab suggest hota hai — wahi kaam Trie karta hai! Trie ek tree hai jismein har node ek letter store karta hai. Root se kisi bhi leaf tak ka path ek complete word banata hai.

Visual Example — How "AAPL", "AMD", "AMZN" are stored:

[ROOT] | [A] ← First letter of all 3 words / \ [A] [M] ← Second letter branches | / \ [P] [D] [Z] ← AMD ends here (is_end=1) | | [L] [N] ← AAPL ends here, AMZN ends here Path ROOT→A→A→P→L = "AAPL" ✓ Path ROOT→A→M→D = "AMD" ✓ Path ROOT→A→M→Z→N = "AMZN" ✓

Key Properties of a Trie

PropertyDescription
Root NodeAlways empty — represents the starting point, no character stored in it
Each EdgeRepresents one character (A-Z in our case = 26 possible children)
Each NodeHas an array of 26 child pointers (one per letter A through Z)
is_end flagMarks whether a complete word ends at this node (1 = yes, 0 = no)
Shared Prefixes"AAPL" and "AMD" share the "A" node — memory efficient for common prefixes!
No HashingUnlike hash tables, tries don't use hash functions — they use character-by-character navigation
⚡ Trie vs Hash Table vs Binary Search Tree

Hash Table: O(1) lookup but CANNOT do prefix search. "A" se start hone wale sab stocks nahi mil sakte.
BST/AVL: O(log n) lookup, can do range queries but not prefix matching efficiently.
Trie: O(m) lookup where m = key length. CAN do prefix search! Type "A" and instantly get all stocks starting with A. This is WHY we use Trie for auto-complete.

Section 02

Why Trie for Stock Symbols?

In a real stock market application, users search for stocks by typing partial symbols. When a user types "A", the system should instantly suggest AAPL, AMD, AMZN, ADBE, AVGO — all symbols starting with "A".

Real-World Use Case

User types: "A" → Suggestions: AAPL, ADBE, AMD, AMZN, AVGO User types: "AA" → Suggestions: AAPL User types: "AAP" → Suggestions: AAPL User types: "AAPL"→ Exact match found! Price: $178.50
