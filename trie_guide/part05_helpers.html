Line 26
TrieNode* n = (TrieNode*)calloc(1, sizeof(TrieNode));
YEH LINE BOHOT IMPORTANT HAI! 🔥 Yahan memory allocate ho rahi hai.

calloc(1, sizeof(TrieNode)) does TWO things:
1. Allocates memory for 1 TrieNode on the HEAP (dynamic memory)
2. Initializes ALL bytes to ZERO — this is KEY!

Why calloc and NOT malloc?
• malloc(sizeof(TrieNode)) — allocates memory but leaves it UNINITIALIZED (garbage values)
• calloc(1, sizeof(TrieNode)) — allocates AND sets everything to 0

Because of calloc:
• All 26 children[] pointers become NULL (0 = NULL for pointers) ✅
• is_end becomes 0 (not an end node) ✅
• last_price becomes 0.0 ✅
• symbol becomes empty string ✅

Without calloc, we'd need 26 lines to manually set each child to NULL!

(TrieNode*) — this is a type cast. calloc returns void* (generic pointer), we cast it to TrieNode* (pointer to TrieNode).

Faculty Answer: "We use calloc instead of malloc because calloc zero-initializes all allocated memory. This automatically sets all 26 child pointers to NULL, is_end to 0, and all other fields to their zero values — eliminating the need for manual initialization of each field."

Line 31: char_idx() — Character to Index Mapping

static int char_idx(char c) { return toupper(c) - 'A'; }
Line 31
static int char_idx(char c) { return toupper(c) - 'A'; }
This is the MAPPING function — converts a letter to an array index (0-25).

How it works step by step:
1. toupper(c) — converts character to uppercase. 'a'→'A', 'B'→'B' (already uppercase stays same)
2. - 'A' — subtracts ASCII value of 'A' (which is 65)

Examples:
• char_idx('A') = toupper('A') - 'A' = 65 - 65 = 0 → children[0]
• char_idx('B') = toupper('B') - 'A' = 66 - 65 = 1 → children[1]
• char_idx('M') = toupper('M') - 'A' = 77 - 65 = 12 → children[12]
• char_idx('Z') = toupper('Z') - 'A' = 90 - 65 = 25 → children[25]

This is a one-liner function — the entire function body is on one line with curly braces. It's equivalent to writing it in 3 lines.

Faculty Answer: "char_idx maps a character to its corresponding array index using ASCII arithmetic. toupper() ensures case-insensitivity, and subtracting 'A' maps the uppercase letter to the range 0-25, which directly indexes into the children array."
💡 ASCII Value Table (Important for Viva)

'A' = 65, 'B' = 66, 'C' = 67 ... 'Z' = 90
'a' = 97, 'b' = 98, 'c' = 99 ... 'z' = 122
'0' = 48, '1' = 49 ... '9' = 57

So 'M' - 'A' = 77 - 65 = 12. This trick works because ASCII values for letters are contiguous (no gaps between A and Z).
