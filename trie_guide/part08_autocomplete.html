Lines 58–76: collect() & get_suggestions() — Auto-Complete

These two functions work TOGETHER to implement auto-complete. get_suggestions navigates to the prefix node, then collect does a DFS to find all complete words below it.

Part A: collect() — Recursive DFS Collector

static void collect(TrieNode* node, char suggestions[][MAX_SYMBOL_LEN], int* cnt, int max) {
    if(*cnt >= max) return;
    if(node->is_end) strcpy(suggestions[(*cnt)++], node->symbol);
    for(int i = 0; i < ALPHABET_SIZE; i++)
        if(node->children[i]) collect(node->children[i], suggestions, cnt, max);
}
Line 58
static void collect(TrieNode* node, char suggestions[][MAX_SYMBOL_LEN], int* cnt, int max) {
Parameters breakdown:
• TrieNode* node — current node being visited in the DFS
• char suggestions[][MAX_SYMBOL_LEN] — a 2D array to store results. Each row is one symbol string. Think of it as a list of strings.
• int* cnt — pointer to a counter (so we can modify it across recursive calls). Tracks how many suggestions found so far.
• int max — maximum number of suggestions to collect (prevents overflow)

Why is cnt a pointer? Because this function is RECURSIVE. If cnt was a regular int, each recursive call would get its own copy and changes wouldn't propagate back. With a pointer, ALL recursive calls share the same counter.

Faculty Answer: "collect is a recursive DFS function. The cnt parameter is passed as a pointer (int*) because the function is recursive and we need all recursive calls to share and modify the same counter. The 2D array suggestions[][MAX_SYMBOL_LEN] stores collected symbol strings."
Line 59
if(*cnt >= max) return;
Base case / Safety check. *cnt dereferences the pointer to get the actual count value. If we've already collected enough suggestions (count >= max), STOP recursing. This prevents array overflow.

Faculty Answer: "This is the termination condition — once we've collected the maximum number of results, we stop the DFS traversal to prevent buffer overflow."
Line 60
if(node->is_end) strcpy(suggestions[(*cnt)++], node->symbol);
DENSE LINE — BREAK IT DOWN! 🔥

1. node->is_end — check if this node marks a complete word
2. If yes: strcpy(suggestions[(*cnt)++], node->symbol)

Breaking down suggestions[(*cnt)++]:
- *cnt — get current count value (e.g., 2)
- suggestions[2] — access row 2 of the 2D array
- strcpy(suggestions[2], node->symbol) — copy the symbol string into that row
- (*cnt)++ — THEN increment count to 3 (post-increment)

So if count is 2 and node has "AMZN", it copies "AMZN" into suggestions[2] and increments count to 3.

Faculty Answer: "When we reach a node with is_end=1, we copy its symbol into the suggestions array at the current count index, then post-increment the count. The expression (*cnt)++ first dereferences the pointer to access the count, uses it as the array index, then increments it."
Lines 61–62
for(int i = 0; i < ALPHABET_SIZE; i++)
if(node->children[i]) collect(node->children[i], suggestions, cnt, max);
THE RECURSION! Loop through all 26 possible children. For each child that EXISTS (not NULL), recursively call collect on it.

This is a Depth-First Search (DFS) — we go deep into each branch before trying the next one. Since we iterate i from 0 to 25 (A to Z), results come back in alphabetical order!

Faculty Answer: "This implements a DFS traversal over all children. By iterating i from 0 (A) to 25 (Z), we naturally collect results in lexicographic (alphabetical) order. The recursive structure explores each subtree completely before moving to the next sibling."

Part B: get_suggestions() — Navigate to Prefix, Then Collect

static int get_suggestions(TrieNode* root, const char* prefix,
                            char suggestions[][MAX_SYMBOL_LEN], int max) {
    TrieNode* cur = root;
    for(int i = 0; prefix[i]; i++) {
        int idx = char_idx(prefix[i]);
        if(idx < 0 || idx >= ALPHABET_SIZE || !cur->children[idx]) return 0;
        cur = cur->children[idx];
    }
    int cnt = 0;
    collect(cur, suggestions, &cnt, max);
    return cnt;
}

Step 2: Collect all words below this prefix node.

• int cnt = 0 — initialize result counter
• &cnt — pass the ADDRESS of cnt (pointer) so collect can modify it
• collect(cur, ...) — start DFS from the prefix node. Finds ALL complete words below
• return cnt — return how many suggestions were found

Two-step process:
1. Navigate: O(m) — walk to the prefix node
2. Collect: O(r) — DFS to collect r results
Total: O(m + r)

Faculty Answer: "get_suggestions implements a two-phase algorithm: first, it navigates to the node representing the end of the prefix in O(m) time, then calls collect() to perform a DFS from that node, gathering all complete words in O(r) time where r is the number of results. The total complexity is O(m + r)."
