Lines 48–56: trie_search() — Exact Lookup

This function checks if a COMPLETE symbol exists in the trie. Returns 1 (true) if found, 0 (false) if not.

static int trie_search(TrieNode* root, const char* symbol) {
    TrieNode* cur = root;
    for(int i = 0; symbol[i]; i++) {
        int idx = char_idx(symbol[i]);
        if(idx < 0 || idx >= ALPHABET_SIZE || !cur->children[idx]) return 0;
        cur = cur->children[idx];
    }
    return cur && cur->is_end;
}

Similar loop as insert, BUT with one key difference on Line 52:

if(idx < 0 || idx >= ALPHABET_SIZE || !cur->children[idx]) return 0;

Three conditions checked with || (OR):
1. idx < 0 — invalid character (below 'A')
2. idx >= ALPHABET_SIZE — invalid character (above 'Z')
3. !cur->children[idx] — the child doesn't exist! This means the word is NOT in the trie.

If ANY of these is true → immediately return 0 (NOT FOUND). No need to continue.

Key difference from insert: In insert, if child is NULL, we CREATE a new node. In search, if child is NULL, we GIVE UP — the word isn't there.

Faculty Answer: "The search loop navigates the trie character by character. If at any point the required child pointer is NULL, it means the search key doesn't exist in the trie, and we return 0 immediately. This is an early termination optimization — we don't need to check the remaining characters."
Line 55
return cur && cur->is_end;
THIS LINE IS TRICKY — FACULTY FAVORITE! 🎯

After the loop, cur points to the node of the last character. But we need TWO checks:

1. cur — is the pointer not NULL? (safety check)
2. cur->is_end — is this node marked as end of a word?

WHY do we need is_end check?
Imagine "AMD" is in the trie. If someone searches for "AM":
- Loop follows A→M successfully ✅
- But cur->is_end is 0 at M node (because "AM" is NOT a valid stock symbol)
- Returns 0 (NOT FOUND) — correct!

Without is_end check, "AM" would falsely return as found just because the path exists.

&& is logical AND — BOTH conditions must be true to return 1.

Faculty Answer: "The final return statement checks two conditions: (1) cur is not NULL (safety), and (2) cur->is_end is 1 (the node marks a complete word). This distinction is crucial — a prefix like 'AM' would navigate the trie successfully but would fail the is_end check since 'AM' is not a stored key, only 'AMD' is."

⏱️ Time Complexity: O(m)

Search takes O(m) time where m = length of the search key. For "GOOGL" (5 chars), exactly 5 steps. This is independent of how many total words are in the trie! Even with 1 million stocks, searching "GOOGL" still takes only 5 steps.
