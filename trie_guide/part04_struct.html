Section 04

Lines 13–24: Data Structure & Global Variables

These lines define THE most important thing — the TrieNode structure and tracking variables.

Line 13: ALPHABET_SIZE constant

#define ALPHABET_SIZE 26
Line 13
#define ALPHABET_SIZE 26
#define is a preprocessor macro. Before compilation, the compiler replaces EVERY occurrence of ALPHABET_SIZE with 26 in the code. It's like a find-and-replace.

Why 26? Because English alphabet has 26 letters (A to Z). Our stock symbols only contain uppercase English letters, so each trie node needs exactly 26 child pointers — one for each letter.

Faculty Answer: "ALPHABET_SIZE is defined as 26 because we're working with uppercase English letters A-Z. Each trie node will have an array of 26 child pointers, one for each possible character. This is a compile-time constant defined using a preprocessor macro."

Lines 15–20: The TrieNode Structure

typedef struct TrieNode {
    struct TrieNode* children[ALPHABET_SIZE];  // 26 child pointers
    int is_end;                              // is this a complete word?
    char symbol[MAX_SYMBOL_LEN];             // full symbol name
    double last_price;                       // stock price
} TrieNode;
Line 15
typedef struct TrieNode {
typedef = "type definition" — lets us create a custom name for a data type. Instead of writing struct TrieNode everywhere, we can just write TrieNode.

struct TrieNode = we're defining a structure (a group of related variables bundled together). The name TrieNode after struct is needed because this struct is self-referential — it contains pointers to itself (children are TrieNode pointers).

Faculty Answer: "This is a self-referential structure definition using typedef. The struct tag 'TrieNode' is necessary because the children array inside the struct needs to reference the same struct type, creating a recursive data structure."
Line 16
struct TrieNode* children[ALPHABET_SIZE];
YEH SABSE IMPORTANT LINE HAI! 🔥

This declares an array of 26 pointers. Each pointer can point to another TrieNode (child node).

children[0] = pointer to child for letter 'A'
children[1] = pointer to child for letter 'B'
children[2] = pointer to child for letter 'C'
...
children[25] = pointer to child for letter 'Z'

If children[i] is NULL, it means no word passes through that letter at this level.
If children[i] is NOT NULL, it points to the next node in the path.

Memory Impact: Each node stores 26 pointers × 8 bytes (64-bit) = 208 bytes per node just for the children array! This is the "space" cost of trie's fast speed.

Faculty Answer: "This is a fixed-size array of 26 child pointers, implementing a 26-ary branching factor. Each index maps to a letter: index 0 for 'A', index 1 for 'B', through index 25 for 'Z'. A NULL pointer at index i means no word continues through that letter from this node. This design enables O(1) child lookup per character but uses more memory than alternatives like linked lists of children."
Line 17
int is_end;
is_end is a flag (0 or 1) that marks whether a complete word ends at this node.

Why is this needed? Consider "AMD" and "AMZN". After inserting "AMD", the node at D has is_end = 1. But the node at M (in the A→M path) has is_end = 0 because "AM" is NOT a valid stock symbol — it's just a prefix.

Without this flag, we couldn't tell if "AM" is an actual word or just a path to other words.

Faculty Answer: "The is_end flag differentiates between nodes that merely form part of a path (prefix) and nodes that mark the completion of a stored key. For example, in the path A→M→D for 'AMD', the 'M' node has is_end=0 (just a prefix) while the 'D' node has is_end=1 (complete word)."
Line 18
char symbol[MAX_SYMBOL_LEN];
Stores the FULL stock symbol string at the end node. When is_end = 1, this field contains the complete symbol like "AAPL" or "MSFT".

MAX_SYMBOL_LEN is defined in stock_data.h (typically 10). So this is a character array that can hold up to 9 characters + 1 null terminator.

Why store the full symbol? Because when we do auto-complete, we collect results from end nodes. Instead of reconstructing the word by walking back up the tree, we simply read node->symbol. Much simpler!

Faculty Answer: "We store the complete symbol string at each end node for efficient retrieval during auto-complete. This avoids the need to reconstruct the word by tracing back from the leaf to the root, trading a small amount of extra storage for simpler collection logic."
Line 19
double last_price;
Stores the stock price as a double (decimal number). Only meaningful at end nodes where is_end = 1.

double = 64-bit floating point number, can store values like 178.50, 875.30, 1295.60 with decimal precision.

Faculty Answer: "The last_price field stores the stock's price at end nodes, making the trie a key-value store — the key is the symbol (encoded in the path), and the value is the associated price data."
Line 20
} TrieNode;
Closes the struct definition. The TrieNode after the closing brace is the typedef name — now we can use TrieNode as a type name directly instead of struct TrieNode.

Lines 22–23: Global Tracking Variables

static int trie_nodes = 0;
static int trie_words = 0;
Lines 22–23
static int trie_nodes = 0;
static int trie_words = 0;
Two counter variables to track trie statistics:

trie_nodes — counts total number of TrieNode objects created (incremented in trie_new_node())
trie_words — counts total number of complete words inserted (incremented in trie_insert())

static keyword: Makes these variables file-scoped — they can only be accessed within this .c file. Other files (like member1_hashtable.c) cannot see or modify these variables. This prevents naming conflicts.

Initialized to 0 — they start fresh and get reset in run_trie().

Faculty Answer: "These are file-scope static variables used for tracking trie statistics. The static keyword limits their visibility to this translation unit, providing encapsulation. trie_nodes tracks total nodes created for memory analysis, and trie_words tracks total keys inserted."
📐 Memory Layout of One TrieNode

children[26] = 26 × 8 bytes = 208 bytes (pointers on 64-bit system)
is_end = 4 bytes (int)
symbol[10] = 10 bytes (char array)
last_price = 8 bytes (double)
Total ≈ 230+ bytes per node (with padding for alignment)

With 58 nodes for 18 stock symbols → approximately 13.3 KB total memory.
